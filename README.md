# 小姐牌（可视化网页）

这是一个基于 **React + TypeScript + Vite** 的“小姐牌”抽牌/规则/状态记录可视化网页项目。

## 操作指南

### 1）环境要求

- Node.js：建议 **Node 20+** 或 **Node 22.12+**
  - 如果你当前是 `v22.11.x`（例如 22.11.0），可能无法运行新版 Vite。此项目已将 Vite 固定在兼容版本以便直接运行。

### 2）安装依赖

在本目录（`web/`）执行：

```bash
npm install
```

### 3）启动开发环境（本地运行）

```bash
npm run dev
```

启动后浏览器打开终端提示的地址（默认是 `http://localhost:5173/`）。

### 4）构建生产包

```bash
npm run build
```

构建产物在 `dist/`。

### 5）本地预览生产包

```bash
npm run preview
```

## 使用说明（游戏界面）

- **抽一张**：从 52 张（无大小王）牌堆中按随机顺序抽牌，抽牌历史会记录（最新在前）。
- **查看规则**：可查看“当前牌/最近抽到/历史记录中任意一张牌”的规则。
- **自动弹窗规则**：
  - 抽到 **2（小姐牌）**：弹窗设置“小姐名”（可为空；最多一个；并且只有抽到 2 才允许修改）。
  - 抽到 **10（神经病）**：弹窗管理“神经病名单”（最多 4 个，可为空，支持修改/删除）。
  - 抽到 **4/5/8**：弹窗管理对应牌的“持牌者”（每种最多 4 个，游戏期间随时可改/删）。
  - 抽到 **K**：弹窗设置“下一个 K 喝多少”，并显示上一次设置的杯数；下次抽到 K 会更新该设置。
- **重置本轮**：清空本轮状态（抽牌历史、小姐名、神经病名单、4/5/8持牌者、K设置），并重新洗牌开始新一轮。

## 常见问题

### Q：运行 `npm run dev` 报 Node/Vite 版本不兼容怎么办？

- 方案 A：升级 Node 到 **22.12+** 或 **20.19+**
- 方案 B：保持当前 Node 版本，但确保依赖已安装成功，然后重新执行：

```bash
rm -rf node_modules package-lock.json
npm install
npm run dev
```
